# 实验三　矩阵乘法应用模拟与分析

## 实验内容

1. 测试单周期处理器与顺序流水线处理器的性能差异
2. 查看随着 CPU 时钟频率的变化所测得的性能如何变化
3. 查看内存带宽和延迟对性能的影响

## 实验开始前的问题

1. 应该使用哪些指标衡量计算机系统的性能?　为什么?

- 从硬件数据指标来说，CPU的时钟频率，各级缓存大小，CPU的核心数量，是决定任务执行速度的最关键指标，CPU频率高，核心数量多，缓存大，内存带宽大，读写延迟低，执行任务的速度就快；

- I/O性能是决定与用户交互的响应速度的关键指标，I/O性能好，调度好，就能够更快的给用户反馈，提高用户的体验；

- 从实际检测指标来说，执行时间/响应时间，吞吐量，每秒执行指令数和每秒浮点计算次数，能耗都是很重要的指标。

- 执行时间是完成任务的所需时间，是最直接最直观的性能指标，执行时间短意味着绝对意义上更好的性能；响应时间是用户提交申请到获得回复所需要的时间，在实时交互的场景中尤为重要，响应时间短意味着用户可以及时收到反馈。

- 吞吐量是指单位时间内完成的任务数量，反映了计算机并行处理能力，直观表现出计算机的整理效率。

- 能耗效率是计算机完成任务消耗的能量，消耗的能量越少，表明计算机在能源成本和散热上的优势。

2. 为什么不总是可以使用相同的性能指标来评估计算机系统？

- 首先是应用场景的不同会有不同的需求，就会有不同的标准。比如说对于个人PC，及时交互，执行速度，响应速度是最需要关注的指标；对于数据中心，I/O交互，能耗可能更加重要；对于科学计算，浮点计算性能会更加重要。

- 系统架构的不同。对于多核高并行的场景，核心数量，缓存大小更加重要；对于非并发高负载，单核心性能更加重要；对于分布式系统，计算和传输可靠性和速度更重要。

- 对于CPU瓶颈和内存瓶颈，需要关注的点就不一样。

## 实验步骤

### 步骤１：更改CPU模型和缓存时钟频率

1.  在相同的时钟频率之下，单周期CPU(HW1TimingSimpleCPU)和顺序流水线CPU(HW1MinorCPU)之间，哪个CPU将表现出更好的性能？为什么？

在相同的时钟频率之下，顺序流水线CPU体现了更好的性能。具体仿真结果如下：

| 时钟频率\仿真时间(ms) | 单周期CPU | 顺序流水线CPU |
| --------------------- | --------- | ------------- |
| 1GHz                  | 287.841   | 133.542       |
| 2GHz                  | 144.143   | 66.9952       |
| 4Ghz                  | 72.2922   | 33.7088       |

原因：在相同的时钟频率之下，流水线CPU将每个指令拆成多个阶段，每个阶段一个周期完成，实现了指令级并行，提高了吞吐率；同时流水线CPU在同一时间执行不同指令的不同阶段，减少了指令之间的延迟；在同一时间，CPU的各个资源同时在工作，减少了资源的浪费和空闲。由此顺序流水线CPU实现了同一时钟频率下的更好的性能。

2. 在单周期 CPU（HW1TimingSimpleCPU）和顺序流水线 CPU（HW1MinorCPU）之间，哪一个对于时钟频率的变化更为敏感？为什么？

单周期CPU对于时钟频率的变化更敏感。从理论上来说，所有的指令都必须在一个周期内完成，周期的变化就会直接影响每条指令的执行速度。但是对于顺序流水线CPU，它可以把每个指令都拆分成不同的阶段，在不同的时钟周期内完成，因此顺序流水线CPU的性能对时钟频率的变化不如单周期CPU敏感。

### 步骤２：更改CPU和内存模型

1. 如果在计算机系统中将内存性能提高一倍（加倍带宽并减半延迟），整体性能是否也会加倍？为什么？

不会使整体性能加倍。执行程序的性能不会完全由内存决定，这其实是一个很复杂的结果，会影响执行时间的因素很多，比如CPU频率限制，分支预测，计算逻辑，网络环境，在不同的实际条件下还会受到瓶颈因素的制约，即使内存性能翻倍，也只能够让被内存制约的一部分过程变快，这部分性能变快也可能受到其他因素的制约，也不可能直接使得整体性能翻倍。

2. 在改善内存性能方面，HW1TimingSimpleCPU 和 HW1MinorCPU 之间哪个CPU 模型将更受益？为什么？

仿真结果如下：

| 内存模型\仿真时间(ms) | 单周期CPU | 顺序流水线CPU |
| --------------------- | --------- | ------------- |
| HW1DDR3_1600_8x8      | 72.2922   | 33.7088       |
| HW1DDR3_2133_8x8      | 72.2635   | 33.6794       |
| HW1LPDDR3_1600_1x32   | 72.5038   | 33.6794       |

可以发现提高内存对单周期CPU影响更明显。因为单周期CPU的访存会阻塞后续指令，无法隐藏内存延迟，所以内存延迟对于执行时间的影响就会比较大，但是顺序式流水线可以部分隐藏内存延迟，所以改善内存对于性能的提升就小一些。

### 步骤３：一般问题：如果你使用不同的应用程序，你认为你的结论会改变吗？为什么？

如果使用不同的程序进行仿真测试，测试的结果一般不会发生根本性的改变，但是不同硬件规格的调整产生的收益可能会发生很大的变化。

原因是不同的应用程序的资源调度，计算负载，对计算机的要求可能是不一样的，影响执行性能的关键因素可能也会不同。

对于计算密集型的程序，CPU的性能会成为主要瓶颈，内存对执行结果的影响可能很小，提高CPU频率对结果的影响可能很大；对于资源密集型的程序，需要频繁提取大量数据，那么提高内存应该会对实际性能的提升产生非常明显的影响。

但是在多数情况下，顺序流水线CPU的性能优于单周期CPU，时钟频率和改善内存对于单周期CPU的性能影响依旧更加明显。

## 心得体会

通过gem5实际模拟不同硬件对于同一个工作负载的执行性能的影响，我了解了单周期CPU和顺序流水线CPU的不同特点，学习了通过不同指标衡量机器对于执行负载的性能分析，分析了不同硬件条件比如CPU的时钟频率和内存等因素对于仿真结果的影响。
